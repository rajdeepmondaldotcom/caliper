# Future 1.0.0 Migration Notes

Caliper 1.0.0 is reserved for the local CLI and JSON compatibility freeze.
The current pre-1.0 line already emits `schema_version: 1`, but the SemVer
contract is not declared frozen until 1.0.

## Compatibility

- Existing CLI commands continue to work.
- Existing JSON consumers should read the top-level `caliper` envelope and
  pin on `schema_version == 1`.
- New fields remain additive inside existing objects.
- Future deprecations will print to stderr with a `[deprecated]` prefix
  before removal.

## New Commands

```bash
caliper schema export --name usage_event --format json
caliper schema validate ./event.json --name usage_event
```

## Required Action

No action is required unless you parse Caliper JSON output directly. JSON
consumers should begin checking `caliper.schema_version`.
