# Security Policy

Caliper reads sensitive local data (AI coding session logs, project paths,
git metadata). It is designed to keep that data on the user's machine.
Vulnerabilities that weaken that posture, that expose user data, or that
enable supply-chain attack against downstream installs are treated as the
highest severity.

## Supported versions

Caliper is pre-1.0. Only the most recent patch release receives security
fixes.

| Version | Supported |
| ------- | --------- |
| `0.0.53` | yes |
| `< 0.0.53` | no |

The `1.0` release will introduce a longer support window and a published
deprecation policy. Until then, please upgrade promptly.

## Reporting a vulnerability

**Please do not file a public GitHub issue for security-impacting bugs.**

Use **[GitHub Private Vulnerability Reporting](https://github.com/rajdeepmondaldotcom/caliper/security/advisories/new)**.
Reports submitted that way are visible only to the project maintainers
and remain private until a fix ships.

If for any reason GitHub PVR is unavailable, open a minimal public
issue titled "security: contact requested" (no details) and a
maintainer will reach out privately.

A good report includes:

1. The Caliper version (`caliper --version`).
2. The operating system and Python version.
3. The exact steps to reproduce, including any local fixtures or commands.
4. The impact assessment (what data is exposed, what guarantee is broken).
5. Any suggested fix or mitigation.

We aim to acknowledge new reports within **72 hours** and to ship or stage
a fix within **14 days** for high-severity issues. Lower severity issues
follow the normal release cadence.

## Scope

In scope:

- The Caliper CLI (`src/caliper/`).
- The PyPI distribution `caliper-ai`.
- The GitHub Actions release pipeline (`.github/workflows/release.yml`).
- The PyPI Trusted Publisher configuration for the protected GitHub Actions
  `pypi` environment.

Out of scope:

- Downstream forks or third-party distributions.
- The accuracy of an external vendor's published pricing data (Caliper
  surfaces our embedded rate card and its `checked` date — staleness is
  not a security issue).
- Theoretical issues with Python, `rich`, `typer`, `platformdirs`,
  `textual`, or `watchdog` that do not have a working reproduction against
  Caliper.

## What "in scope" looks like in practice

Examples of vulnerability classes we want reported:

- Unintended exfiltration of session contents, prompts, or paths through
  any command, format, or export.
- Bypass of the explicit network boundary. Caliper-owned HTTP I/O lives in
  `caliper/network.py`; its callers (`caliper/pricing_catalog.py`,
  `caliper/rate_audit.py`) are guarded by `--allow-network` at the CLI
  boundary. PR commit resolution may delegate to GitHub CLI only when the
  user passes `--allow-network`. Any code path that reaches the network on a
  default invocation is in scope.
- Code execution via untrusted JSONL, SQLite, or chat-history fixtures.
- Supply-chain compromise of the PyPI distribution.
- Pricing math that silently produces wrong numbers (treated as
  data-integrity, security-adjacent).
- Privacy regressions in receipts, JSON exports, or live TUI output
  (e.g. prompts leaking past `REDACTION_LIMIT`).

## Disclosure policy

Once a fix has shipped, we publish a GitHub Security Advisory with:

- CVE (if assigned).
- Affected versions.
- Mitigations available before upgrading.
- Credit to the reporter (if they wish to be named).

We coordinate disclosure timelines with reporters and never name a
reporter publicly without their consent.

## Cryptographic verification of releases

- Every release wheel and sdist on PyPI is published from GitHub Actions
  using the `PYPI_API_TOKEN` secret scoped to the protected `pypi`
  environment. Keep that token project-scoped to `caliper-ai` on PyPI and
  rotate it after any suspected exposure.
- Every release artifact carries a **SLSA build provenance attestation**
  generated by `actions/attest-build-provenance@v4`.
- Every release ships a **CycloneDX SBOM** alongside the wheel.

Verification:

```bash
gh attestation verify --owner rajdeepmondaldotcom dist/caliper_ai-<version>-py3-none-any.whl
```

## Thank you
